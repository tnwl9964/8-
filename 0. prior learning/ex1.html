WEC접근성 기준 개요

소개

월드 와이드 웹 컨소시움(W3C)은 국제 웹 표준을 만들었습니다 : HTML, CSS을 몇 가지가 있습니다. W3C의 웹 표준을 W3C Recommendatons이라 부릅니다.

모든 W3C 표준을 접근성 지원을 위해 접근성 플렛폼 설계(APA(영어))실무 그룹에서 검토합니다.

아래 소개된 W3C 표준과 실무 그룹의 기록은 특히 접근성과 있습니다.

접근성 지침

웹 접근성 필수요소는 웹 접근성이 웹 개발 및 상호작용 과정에서

여러요소와 어떻게 작용하는지와 WAI 지침(WEAG, ATAG, UAAC)의 적용방법에 대해 명시하고 있습니다.

웹 콘텐츠 접근성 지침 (wcag) 2

웹 "콘텐츠"는 일반적으로 웹 페이지나 어플리케이션 안의 정보를 가리킵니다.

WCAG는 다양한 콘텐츠, 멀티미디어, "모바일" 등에 적용됩니다.

WCAG는 WCAG21CT (영어)에서 명시하듯이 웹이 아닌 정보 통신 기술(ICT)에도 적용됩니다.

웹 저작 도구 접근섭 지침 (ATAG)

저작도구는 "저자" (웹 개발자, 디자이너, 작가 등)가 웹 콘텐츠를 생산하는 데에 사용하는 서비스나 소프트웨어를 가리킵니다.예를 들어, HTML 에디터,

콘텐츠 관리 시스템 (CMS) 그리고 사용자가 콘텐츠를 생산항 수 있는 블로그나 소셜 네트워킹 사이트가 있습니다.

사용자 에이전트 접극성 지침 (UAAC)

사용자 에이전트는 브라우저, 브라우저 확장프로그램, 미디어 프레이어, 리더디와 같은 웹 콘텐츠를 제공하는 어플리케이션을 포함합니다.

기술 명세

접근가능한 리치인터넷 어플리케이션 (WAL-ARIA)

ARIA는 의미를 제공하여 저자가 유저 인터페이스 행동이나 구조적 정보를 스크린 리더와 같은 보조 기술에 전달할 수 있게 합니다. ARIA 기술 명세는 역할, 상태, 접근 가능한 유저 인터페이스 요소로 정의되는 속성들의 온톨로지를 제공합니다.

ARIA 모음은 유저 에이전트 실행 지침을 제공하는 명세를 모은 API를 포함합니다. 그래픽, 디지털 출판을 위한 모듈 또힌 포함됩니다.
